# 実現可能性検証: 営業マン別無料枠戦略

**調査日**: 2026年1月2日
**目的**: 顧客向け文字起こしアプリをGemini API無料枠で実現できるか検証

---

## ビジネスモデル

1. 法人を設立
2. 法人の顧客向けに文字起こしアプリを無償提供
3. 顧客は自社の営業マンごとにGemini APIの無料枠を活用
4. 録音した商談や会議の文字起こし・分析を実質無料で実現

## 検証1: 営業マン別アカウントで無料枠は増えるか？

### 結論: Yes、増える

| ポイント | 内容 |
|---------|------|
| 制限単位 | **プロジェクト単位**（APIキー単位ではない） |
| 複数プロジェクト | 各プロジェクトが独立したクォータを持つ |
| 実現方法 | 営業マン1人につき1 Google Cloudプロジェクトを作成 |

### 重要な仕様

- レート制限はプロジェクト単位で適用
- 同一プロジェクト内で複数APIキーを作っても無料枠は増えない
- 別プロジェクトを作れば、それぞれ独立した無料枠が付与される

## 検証2: 音声→テキスト変換の選択肢（2026年1月時点）

### Option A: Google Cloud Speech-to-Text API

| 項目 | 内容 |
|------|------|
| 無料枠 | **60分/月**（全ユーザー共通） |
| 新規顧客 | $300クレジット（90日間）→ 約18,750分相当 |
| 有料 | $0.016/分（標準）、$0.004/分（バッチ） |
| 評価 | 無料枠が少なく、本ユースケースには不向き |

### Option B: Gemini API（音声入力・マルチモーダル）【推奨】

| 項目 | 内容 |
|------|------|
| 機能 | 音声ファイルを直接入力し、文字起こし＋分析を一括処理 |
| トークン換算 | **32トークン/秒**（1時間 = 115,200トークン） |
| 無料枠（Flash-Lite） | 1,000-1,500 RPD、250,000 TPM |
| 評価 | 本ユースケースに最適 |

### Gemini APIの音声処理について

- Geminiはマルチモーダルモデルとして音声入力をサポート
- 音声ファイルをアップロードし、1リクエストで文字起こし＋分析が可能
- リアルタイム文字起こしは非対応（録音後の処理に使用）

## 無料枠での実現可能性

### Gemini 2.5 Flash-Lite使用時

| 項目 | 1日の使用量 | 無料枠 | 判定 |
|------|------------|--------|------|
| RPD | 約8リクエスト（4商談×2） | 1,000-1,500 | **余裕** |
| TPM | 115,200トークン/商談 | 250,000 | **1商談ずつなら問題なし** |

### 計算詳細

- 1商談（1時間）の音声 = 3,600秒 × 32トークン = 115,200トークン
- TPM制限（250,000）以内なので、1商談ずつ処理すれば問題なし
- 1日4商談 = 約8リクエスト（音声入力4 + 追加分析4）
- RPD制限（1,000-1,500）に対して十分余裕あり

### 結論

**営業マン1人につき1プロジェクトを作れば、無料枠内で1日4商談は処理可能**

## 推奨アーキテクチャ

```
【顧客企業】
    │
    ├─ [営業マンA] → [GCPプロジェクトA] → Gemini 2.5 Flash-Lite
    │       ↓              ↓
    │     音声録音    無料枠: 1,000 RPD
    │       ↓
    │     文字起こし＋分析（1リクエストで完結）
    │
    ├─ [営業マンB] → [GCPプロジェクトB] → Gemini 2.5 Flash-Lite
    │       ↓              ↓
    │     音声録音    無料枠: 1,000 RPD（別枠）
    │
    └─ [営業マンC] → [GCPプロジェクトC] → ...
```

### セットアップフロー（営業マンごと）

1. Google Cloudアカウント作成（または既存アカウント使用）
2. 新規プロジェクト作成
3. Gemini APIを有効化
4. APIキー発行
5. アプリにAPIキーを設定

## リスクと対策

### リスク1: 無料枠の突然の削減

| 内容 | 対策 |
|------|------|
| 2025年12月7日にGoogleが予告なく無料枠を92%削減した実績あり | 有償化した場合のコスト（約200円/月）を顧客に事前説明 |

### リスク2: ToS違反の懸念

| 内容 | 対策 |
|------|------|
| 無料枠の過度な活用がToS違反と見なされる可能性 | Cloud Billing有効化を推奨（$0でもBilling設定で正規利用扱い） |

### リスク3: モデル廃止・変更

| 内容 | 対策 |
|------|------|
| Flash-Liteが廃止・制限強化される可能性 | 複数モデルに対応できる設計にしておく |
